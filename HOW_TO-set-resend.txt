# HOW TO - Configurar Resend per als Magic Links de Selectora

Aquest document explica com configurar **Resend SMTP** perquè Selectora pugui enviar els emails de login amb **magic link**.

## Objectiu

Quan un usuari entra el seu email al formulari de login:

1. Selectora crea un magic link.
2. Django envia l'email via Resend SMTP.
3. L'usuari clica l'enllaç.
4. L'usuari queda autenticat.

## 1. Verificar el domini a Resend

A Resend, afegeix i verifica el domini des del qual vols enviar emails.

Exemple:

```text
selectora.cc
```

Resend mostrarà registres DNS que cal afegir al DNS del domini.

Espera fins que Resend indiqui que el domini està verificat.

## 2. Crear una API key de Resend

A Resend, crea una API key per enviar emails.

Tindrà un format semblant a aquest:

```text
re_xxxxxxxxxxxxxxxxxxxxxxxxx
```

> No facis commit de la clau real al repositori.

## 3. Configurar variables d'entorn en producció

Al servidor de producció, defineix aquestes variables:

```bash
EMAIL_BACKEND=django.core.mail.backends.smtp.EmailBackend
EMAIL_HOST=smtp.resend.com
EMAIL_PORT=587
EMAIL_HOST_USER=resend
EMAIL_HOST_PASSWORD=re_xxxxxxxxxxxxxxxxxxxxxxxxx
EMAIL_USE_TLS=true
EMAIL_USE_SSL=false
EMAIL_TIMEOUT=10
DEFAULT_FROM_EMAIL=Selectora <noreply@selectora.cc>
```

Notes importants:

- `EMAIL_HOST_USER` ha de ser sempre `resend`.
- `EMAIL_HOST_PASSWORD` és la API key de Resend.
- `DEFAULT_FROM_EMAIL` ha d'usar un domini verificat a Resend.

## 4. Reiniciar l'aplicació Django

Després de canviar variables d'entorn, reinicia el procés que executa Django.

Exemple amb systemd:

```bash
systemctl restart selectora
```

O reinicia el contenidor/process manager que estigui executant Django.

## 5. Provar l'enviament des de Django shell

Al servidor de producció, amb l'entorn carregat, obre el shell:

```bash
.venv/bin/python manage.py shell
```

Després executa:

```python
from django.core.mail import send_mail

send_mail(
    "Selectora email test",
    "If this arrives, Resend SMTP is configured.",
    None,
    ["your-email@example.com"],
    fail_silently=False,
)
```

Si SMTP falla, Django mostrarà una excepció perquè `fail_silently=False`.

## 6. Provar el flux real de magic link

1. Obre Selectora.
2. Ves al login.
3. Escriu un email.
4. Resol el captcha.
5. Envia el formulari.

Resultat esperat:

- La pàgina redirigeix a la pantalla de revisar l'email.
- El destinatari rep el magic link.
- Clicar el magic link inicia sessió.

## Referència ràpida

| Opció | Valor |
| --- | --- |
| SMTP host | `smtp.resend.com` |
| SMTP username | `resend` |
| SMTP password | API key de Resend |
| Port STARTTLS | `587` |
| Port SSL | `465` |
| From email | Domini verificat a Resend |

## Configuració TLS / SSL

Per port `587`:

```bash
EMAIL_USE_TLS=true
EMAIL_USE_SSL=false
```

Per port `465`:

```bash
EMAIL_USE_TLS=false
EMAIL_USE_SSL=true
```

## Notes de diagnòstic

- Si no arriben emails, revisa els logs de Django.
- Si el domini no està verificat, Resend pot rebutjar l'enviament.
- Si `DEFAULT_FROM_EMAIL` no usa el domini verificat, l'enviament pot fallar.
- Django ja envia magic links amb `fail_silently=False`, per tant els errors SMTP han de quedar visibles als logs.
